Founder Software

Founder Inbox.

A minimalist unified mail and task manager leveraging automated summary threads to prioritize investor relationships and customer inquiries.

View GitHub Repository

Overview

Early-stage startup founders negotiate investor funding, manage client pipelines, triage bug reports, and run internal teams. Inboxes get flooded with newsletters, notifications, and updates, directly burying high-value conversations. We built Founder Inbox to separate operations noise from business leverage.

Project Type

Founder Software

Client State

Active Beta Release

The Problem

Founders waste time managing email. Standard email clients display messages chronologically without priority weight. Important client follow-ups are easily missed in the daily clutter. Delayed responses directly impact conversion cycles, while copying to-do lists out of email creates manual friction.

Discovery Workshop

We found that triage speed is driven by visual density and keyboard interaction controls. Incorporating a dual-pane split view, and keyboard keybindings to archive/schedule tasks, gives founders a highly optimized interface to reach inbox zero.

Product Thinking

Standard email tools overload users with features. We built a hyper-focused triage layer that displays thread summary notes first. Removing folder trees and buttons, and relying on fast VIM hotkeys, converts email reading into active execution task clearance.

Design & wireframes

We designed a high-density, terminal-inspired email list. It keeps layouts clean, using simple monospace codes and labels to represent thread priorities instead of corporate graphics details.

System Architecture

The inbox runs an active database listener connected to Google Workspace and Outlook APIs. Ingested headers pass through a lightweight classification model, sorting messages into priority categories (Investors, Leads, Support). An LLM summary pipeline runs asynchronously to outline key tasks and target actions.

Tech Stack

React.js Node.js Supabase DB OpenAI API Gmail Webhooks IMAP / SMTP Vite Core

Key Features

  • ⚡ Dual-Pane Split Layout: Side-by-side message details and action summarizations.
  • ⚡ Keyboard Triage Controls: VIM-style hotkeys to categorize and clear.
  • ⚡ Investor Smart Folder: Automatically flags and highlights funding channels.
  • ⚡ Action Summaries: Highlights core todo tasks and deadlines instantly.

Engineering Challenges

Real-time email synchronization via webhooks has latency and throttling constraints. Gmail limits API call rates under load. We solved this by implementing a Redis cache queue that aggregates and processes events, updating the client DB and local states asynchronously.

Outcome

Beta validation saw a 3x increase in triage speeds. Over 9.4% more high-value client follow-up meetings were booked, while founders reduced email management times by 65%.

Future Roadmap

We are configuring automated draft generators. The tool will parse past emails to learn writing styles, drafting customized context responses with a single keystroke.
