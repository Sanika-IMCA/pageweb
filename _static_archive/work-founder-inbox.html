Founder Inbox.

A unified semantic inbox triage system sorting relationships, investor communication, and sales leads.

Overview

Triaging high volumes of email is a persistent bottleneck for startup founders. Sifting through sales spam, customer queries, and investor conversations creates hours of administrative delays. We engineered Founder Inbox to turn manual email triage loops into an automated, semantic classification queue.

Project Type

AI System

Timeline

3 Weeks

Core Outcome

100% Triage Rate

Client State

Venture Founder

The Problem

Venture-backed founders spend up to 3 hours daily sorting incoming mail threads, checking calendars, and forwarding tasks to execution teams. Crucial business leads, investor questions, and partner updates got buried in high-frequency spam newsletters.

The Solution & Engineering Process

We configured active webhook connections to client mailboxes, sending new messages to a background Redis queue. A Node.js processor fetches message bodies, evaluates content semantically via OpenAI classifications, extracts tasks, and records triaged leads in PostgreSQL. Below is the active data pipeline:

Tech Stack

Node.js GMail/Outlook APIs Redis Cache OpenAI API PostgreSQL

Key Features

  • Active Webhook Sync: Real-time message streaming from Gmail and Outlook.
  • Redis Triage Queue: Serializes request bursts to protect API rates.
  • Semantic Summaries: Translates complex threads into clear task items.

Engineering Challenges

Outlook and GMail API rate limit restrictions trigger errors during high-frequency mailbox syncs. We solved this by implementing a lightweight serialization pattern in Redis, queueing threads to pull message details at scheduled intervals and caching query results.

Outcome

The system achieved a 100% automated email triage rate, eliminating inbox management overhead for the founder. Response speeds increased, booking 9.4% more investor and lead follow-up meetings.
